# Biome Accessibility Rules (a11y)

Rules for accessible JSX/React components.

## noAccessKey

Don't use accessKey prop.

```tsx
// BAD - inconsistent across browsers
<button accessKey="s">Save</button>

// GOOD - rely on standard navigation
<button>Save</button>
```

## noAriaHiddenOnFocusable

Don't hide focusable elements from screen readers.

```tsx
// BAD
<button aria-hidden="true">Click</button>

// GOOD
<button>Click</button>
<div aria-hidden="true">Decorative</div>
```

## noAriaUnsupportedOnRole

Don't use unsupported ARIA attributes.

```tsx
// BAD
<img role="presentation" alt="description" />

// GOOD
<img role="presentation" alt="" />
```

## noAutofocus

Avoid autofocus (disrupts screen readers).

```tsx
// BAD
<input autoFocus />

// GOOD
<input />
// Focus programmatically when appropriate
```

## noBlankTarget

Use rel="noreferrer" with target="_blank".

```tsx
// BAD - security risk (tabnabbing)
<a href={url} target="_blank">Link</a>

// GOOD
<a href={url} target="_blank" rel="noreferrer">Link</a>
```

## noDistractingElements

Don't use blink/marquee.

```tsx
// BAD
<blink>Blinking text</blink>
<marquee>Scrolling text</marquee>

// GOOD - use CSS animations if needed
<span className="highlight">Important text</span>
```

## noHeaderScope

Scope attribute only on th elements.

```tsx
// BAD
<td scope="row">Data</td>

// GOOD
<th scope="row">Header</th>
<td>Data</td>
```

## noInteractiveElementToNoninteractiveRole

Don't make interactive elements non-interactive.

```tsx
// BAD
<button role="presentation">Click</button>

// GOOD
<button>Click</button>
<div role="presentation">Decorative</div>
```

## noNoninteractiveElementToInteractiveRole

Don't make non-interactive elements interactive via role.

```tsx
// BAD
<div role="button">Click me</div>

// GOOD
<button>Click me</button>
// Or use proper accessibility
<div 
  role="button"
  tabIndex={0}
  onKeyDown={handleKeyDown}
  onClick={handleClick}
>
  Click me
</div>
```

## noNoninteractiveTabindex

Don't add tabindex to non-interactive elements.

```tsx
// BAD
<div tabIndex={0}>Text</div>

// GOOD
<button>Interactive</button>
<div>Non-interactive</div>
```

## noPositiveTabindex

Don't use positive tabindex values.

```tsx
// BAD - breaks natural tab order
<button tabIndex={2}>Second</button>
<button tabIndex={1}>First</button>

// GOOD - use DOM order
<button>First</button>
<button>Second</button>
```

## noRedundantAlt

Don't say "image" in alt text.

```tsx
// BAD
<img alt="Image of a cat" />
<img alt="Photo: sunset" />
<img alt="Picture of dog" />

// GOOD
<img alt="A cat sleeping" />
<img alt="Sunset over ocean" />
<img alt="Golden retriever playing" />
```

## noRedundantRoles

Don't specify implicit roles.

```tsx
// BAD - nav already has navigation role
<nav role="navigation">...</nav>
<button role="button">Click</button>
<a href="#" role="link">Link</a>

// GOOD
<nav>...</nav>
<button>Click</button>
<a href="#">Link</a>
```

## noSvgWithoutTitle

SVGs need accessible names.

```tsx
// BAD
<svg>
  <circle />
</svg>

// GOOD
<svg aria-label="Circle icon">
  <circle />
</svg>

// Or with title
<svg>
  <title>Circle icon</title>
  <circle />
</svg>

// Decorative SVG
<svg aria-hidden="true">
  <circle />
</svg>
```

## useAltText

Images need alt attribute.

```tsx
// BAD
<img src="cat.jpg" />

// GOOD
<img src="cat.jpg" alt="A fluffy orange cat" />

// Decorative image
<img src="decoration.jpg" alt="" />
```

## useAnchorContent

Links need accessible content.

```tsx
// BAD
<a href="/about"></a>
<a href="/about"><img src="icon.png" /></a>

// GOOD
<a href="/about">About Us</a>
<a href="/about">
  <img src="icon.png" alt="About Us" />
</a>
<a href="/about" aria-label="About Us">
  <img src="icon.png" alt="" />
</a>
```

## useAriaActivedescendantWithTabindex

Active descendant needs tabindex.

```tsx
// BAD
<div aria-activedescendant="item1">...</div>

// GOOD
<div aria-activedescendant="item1" tabIndex={0}>...</div>
```

## useAriaPropsForRole

Roles need required ARIA attributes.

```tsx
// BAD - checkbox needs aria-checked
<div role="checkbox">Option</div>

// GOOD
<div role="checkbox" aria-checked={checked}>Option</div>
// Or use native
<input type="checkbox" />
```

## useButtonType

Buttons need explicit type.

```tsx
// BAD - defaults to submit in forms
<button>Click</button>

// GOOD
<button type="button">Click</button>
<button type="submit">Submit</button>
<button type="reset">Reset</button>
```

## useFocusableInteractive

Interactive elements must be focusable.

```tsx
// BAD
<div onClick={handleClick}>Clickable</div>

// GOOD
<button onClick={handleClick}>Clickable</button>
<div 
  role="button" 
  tabIndex={0} 
  onClick={handleClick}
  onKeyDown={handleKeyDown}
>
  Clickable
</div>
```

## useHeadingContent

Headings need content.

```tsx
// BAD
<h1></h1>
<h2><img src="logo.png" /></h2>

// GOOD
<h1>Page Title</h1>
<h2><img src="logo.png" alt="Company Name" /></h2>
```

## useHtmlLang

HTML element needs lang attribute.

```tsx
// BAD
<html>...</html>

// GOOD
<html lang="en">...</html>
```

## useIframeTitle

Iframes need title.

```tsx
// BAD
<iframe src="..." />

// GOOD
<iframe src="..." title="Video player" />
```

## useKeyWithClickEvents

Click handlers need keyboard handlers.

```tsx
// BAD
<div onClick={handleClick}>Clickable</div>

// GOOD
<button onClick={handleClick}>Clickable</button>
// Or
<div 
  onClick={handleClick}
  onKeyDown={(e) => e.key === 'Enter' && handleClick(e)}
  tabIndex={0}
  role="button"
>
  Clickable
</div>
```

## useKeyWithMouseEvents

Mouse events need keyboard alternatives.

```tsx
// BAD
<div onMouseOver={show} onMouseOut={hide}>Hover me</div>

// GOOD
<div 
  onMouseOver={show}
  onMouseOut={hide}
  onFocus={show}
  onBlur={hide}
  tabIndex={0}
>
  Hover or focus me
</div>
```

## useMediaCaption

Media needs captions.

```tsx
// BAD
<video src="video.mp4" />
<audio src="audio.mp3" />

// GOOD
<video src="video.mp4">
  <track kind="captions" src="captions.vtt" />
</video>
```

## useValidAnchor

Anchors need valid href.

```tsx
// BAD
<a href="#">Click</a>
<a href="javascript:void(0)">Click</a>

// GOOD
<a href="/page">Navigate</a>
<button onClick={handleClick}>Click</button>
```

## useValidAriaProps

Use valid ARIA attributes.

```tsx
// BAD
<div aria-fake="value">...</div>

// GOOD
<div aria-label="Description">...</div>
```

## useValidAriaRole

Use valid ARIA roles.

```tsx
// BAD
<div role="fake">...</div>

// GOOD
<div role="button">...</div>
<div role="navigation">...</div>
```

## useValidAriaValues

Use valid ARIA values.

```tsx
// BAD
<div aria-hidden="yes">...</div>

// GOOD
<div aria-hidden="true">...</div>
```

## useValidLang

Use valid language codes.

```tsx
// BAD
<html lang="english">...</html>

// GOOD
<html lang="en">...</html>
<html lang="en-US">...</html>
```
