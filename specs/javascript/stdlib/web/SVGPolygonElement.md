# SVGPolygonElement

 Baseline  Widely available 

 This feature is well established and works across many devices and browser versions. It’s been available across browsers since ⁨July 2015⁩. 

- [Learn more](/en-US/docs/Glossary/Baseline/Compatibility)
- [See full compatibility](#browser_compatibility)
- [Report feedback](https://survey.alchemer.com/s3/7634825/MDN-baseline-feedback?page=%2Fen-US%2Fdocs%2FWeb%2FAPI%2FSVGPolygonElement&level=high)

The `SVGPolygonElement` interface provides access to the properties of [<polygon>](/en-US/docs/Web/SVG/Reference/Element/polygon) elements, as well as methods to manipulate them.

## [Instance properties](#instance_properties)

This interface also inherits properties from its parent, [SVGGeometryElement](/en-US/docs/Web/API/SVGGeometryElement).

[SVGPolygonElement.animatedPoints](/en-US/docs/Web/API/SVGPolygonElement/animatedPoints)Read only

An [SVGPointList](/en-US/docs/Web/API/SVGPointList) representing the animated value of the element's [points](/en-US/docs/Web/SVG/Reference/Attribute/points) attribute. If the [points](/en-US/docs/Web/SVG/Reference/Attribute/points) attribute is not being animated, it contains the same value as the `points` property.

[SVGPolygonElement.points](/en-US/docs/Web/API/SVGPolygonElement/points)

An [SVGPointList](/en-US/docs/Web/API/SVGPointList) representing the base (i.e., static) value of the element's [points](/en-US/docs/Web/SVG/Reference/Attribute/points) attribute. Modifications via the [SVGPointList](/en-US/docs/Web/API/SVGPointList) object are reflected in the [points](/en-US/docs/Web/SVG/Reference/Attribute/points) attribute, and vice versa.

## [Instance methods](#instance_methods)

This interface doesn't implement any specific methods, but inherits methods from its parent, [SVGGeometryElement](/en-US/docs/Web/API/SVGGeometryElement).
