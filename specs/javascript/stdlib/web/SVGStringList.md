# SVGStringList

 Baseline  Widely available 

 This feature is well established and works across many devices and browser versions. It’s been available across browsers since ⁨July 2015⁩. 

- [Learn more](/en-US/docs/Glossary/Baseline/Compatibility)
- [See full compatibility](#browser_compatibility)
- [Report feedback](https://survey.alchemer.com/s3/7634825/MDN-baseline-feedback?page=%2Fen-US%2Fdocs%2FWeb%2FAPI%2FSVGStringList&level=high)

The `SVGStringList` interface defines a list of strings.

An `SVGStringList` object can be designated as read only, which means that attempts to modify the object will result in an exception being thrown.

An `SVGStringList` object is indexable and can be accessed like an array.

## [Instance methods](#instance_methods)

[appendItem()](/en-US/docs/Web/API/SVGStringList/appendItem)

Inserts a new item at the end of the list.

[clear()](/en-US/docs/Web/API/SVGStringList/clear)

Clears all existing items from the list, with the result being an empty list.

[initialize()](/en-US/docs/Web/API/SVGStringList/initialize)

Clears all existing items from the list and re-initializes the list to hold the single item specified by the parameter.

[getItem()](/en-US/docs/Web/API/SVGStringList/getItem)

Returns the specified item from the list.

[insertItemBefore()](/en-US/docs/Web/API/SVGStringList/insertItemBefore)

Inserts a new item into the list at the specified position.

[removeItem()](/en-US/docs/Web/API/SVGStringList/removeItem)

Removes an existing item from the list.

[replaceItem()](/en-US/docs/Web/API/SVGStringList/replaceItem)

Replaces an existing item in the list with a new item.
