# PeriodicWave

 Baseline  Widely available 

 This feature is well established and works across many devices and browser versions. It’s been available across browsers since ⁨July 2015⁩. 

- [Learn more](/en-US/docs/Glossary/Baseline/Compatibility)
- [See full compatibility](#browser_compatibility)
- [Report feedback](https://survey.alchemer.com/s3/7634825/MDN-baseline-feedback?page=%2Fen-US%2Fdocs%2FWeb%2FAPI%2FPeriodicWave&level=high)

The `PeriodicWave` interface defines a periodic waveform that can be used to shape the output of an [OscillatorNode](/en-US/docs/Web/API/OscillatorNode).

`PeriodicWave` has no inputs or outputs; it is used to define custom oscillators when calling [OscillatorNode.setPeriodicWave()](/en-US/docs/Web/API/OscillatorNode/setPeriodicWave). The `PeriodicWave` itself is created/returned by [BaseAudioContext.createPeriodicWave](/en-US/docs/Web/API/BaseAudioContext/createPeriodicWave).

## [Constructor](#constructor)

[PeriodicWave()](/en-US/docs/Web/API/PeriodicWave/PeriodicWave)

Creates a new `PeriodicWave` object instance using the default values for all properties. If you wish to establish custom property values at the outset, use the [BaseAudioContext.createPeriodicWave](/en-US/docs/Web/API/BaseAudioContext/createPeriodicWave) factory method instead.

## [Instance properties](#instance_properties)

None; also, `PeriodicWave` doesn't inherit any properties.

## [Instance methods](#instance_methods)

None; also, `PeriodicWave` doesn't inherit any methods.

## [Example](#example)

See [BaseAudioContext.createPeriodicWave](/en-US/docs/Web/API/BaseAudioContext/createPeriodicWave) for simple example code that shows how to create a `PeriodicWave` object containing a simple sine wave.
